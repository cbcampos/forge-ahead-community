Accessibility & Language Guide

Build accessibility and respectful communication into the site from the start.

Forge AHEAD aims to make community information understandable, usable, and easier to access across the site’s resources, programs, and participation pathways.

Overview / Accessibility & Language Guide
Accessibility

Accessibility support and alternate formats

  • Alternate formats for downloadable materials
  • Clear captions, transcripts, or text alternatives when possible
  • Plain-language summaries for technical content
  • Ways to ask for help when a page or file is hard to use
Language Guide

Respectful, plain-language communication

Forge AHEAD aims to use clear, respectful language and make community-facing materials easier to understand across different audiences and settings.

A

Use plain language first

Lead with the clearest explanation possible before adding technical terms, program names, or research language.

B

Design for multiple formats

Assume some visitors will need PDF alternatives, transcripts, captions, or other ways to access information.

C

Explain next steps clearly

Forms, downloads, and participation opportunities explain what happens next and who will follow up.

D

Give visitors a help path

Important pages include clear ways to ask for clarification, alternate formats, or additional support.
